# 연구 조사 프롬프트 — 오픈 라이선스 기반 실시간 얼굴 카툰화 비식별화(De-identification)

> 이 프롬프트를 딥리서치 도구/LLM에 그대로 입력해 모델·방법론을 조사한다.
> **최우선 제약은 라이선스**다. 라이선스가 막힌 후보는 아무리 품질이 좋아도 제외한다.

---

## 1. 배경 · 목표

영상 속 사람 얼굴을 **비식별화(de-identification, 프라이버시 보호)** 하기 위해, 일정 크기 이상의 얼굴을 **카툰/반실사(2.5D) 스타일로 변환**해 원본 인물을 재식별할 수 없게 만드는 파이프라인을 **자체 개발**한다.

- 이미 보유: **YOLOX 기반 파인튜닝 얼굴 검출 모델** (얼굴 위치·크기 검출)
- 전체 흐름:
  `영상 입력 → YOLOX 얼굴 검출 → 크기 임계값 이상 얼굴 선별 → 카툰 스타일화(비식별화) → 원본 프레임에 합성 → 영상 출력`

---

## 2. 절대 제약 (가장 중요, 위반 시 후보 탈락)

- **라이선스: 코드와 사전학습 가중치가 모두 Apache 2.0 또는 MIT여야 한다.**
  - 제외 대상: OpenRAIL, Fair AI Public License, S-Lab License, CC-BY-NC, 커스텀 비상업, "academic/non-commercial only", 상용 API/클로즈드 서비스.
  - 학습에 쓰는 데이터셋도 **상업적 사용이 허용**되어야 한다.
  - 라이선스가 불명확하면 반드시 "확인 필요"로 표기.

---

## 3. 요구사항

- **입출력:** 영상 in → 영상 out.
- **속도(하드 요구):** 1분 영상을 2분 이내 처리 = **≤ 2× 실시간**. 하드웨어 기준 **NVIDIA L4 (24GB VRAM) 단일 GPU**.
- **비식별화 목적:** 결과 얼굴은 **원본 인물로 재식별 불가**해야 함(얼굴 인식 임베딩 유사도가 충분히 낮아야). 단, **자연스럽고 위화감 없어야** 함.
- **유틸리티 보존:** 표정·시선·입 움직임·머리 포즈는 원본과 **일치**해야 함(영상 맥락 유지). ※ 신원은 지우되 표정/동작은 유지 — 이 둘의 분리가 핵심.
- **시간적 일관성:** 프레임 간 깜빡임(flicker) 최소.
- **화풍:** 반실사 2.5D — 커뮤니티 명칭 "Semi-Realistic Anime / Anima" 계열(실사 비율 + 매끈한 일러스트 렌더).
- **자체 호스팅:** 상용 API(DomoAI 등) 불가.
- **얼굴 영역만:** 배경/몸은 실사 유지, 검출된 얼굴 영역만 변환 후 자연스럽게 합성.

## 4. 조사할 핵심 질문

1. **오픈 라이선스(Apache/MIT) 얼굴 스타일화/카툰화 모델** 중 실시간 추론이 가능한 것은? (GAN 계열, 경량 네트워크 포함) 각 후보의 정확한 라이선스(코드+가중치) 확인.
2. **얼굴 비식별화(face de-identification / anonymization) 방법론** 중, **표정·포즈는 유지하고 신원만 제거**하는 최신 기법과 오픈 라이선스 구현체는? (identity-obfuscation, attribute-preserving anonymization 문헌 포함)
3. **카툰 스타일화 자체가 비식별 효과를 주는가?** 스타일화 후 얼굴 인식 유사도가 얼마나 떨어지는지에 대한 연구/벤치마크가 있는가? 비식별화를 보장하려면 스타일화에 무엇을 추가해야 하는가?
4. **실시간 영상에서 시간적 일관성** 확보 방법(오픈 라이선스): optical flow 기반 스무딩, 시간적 손실 학습, 랜드마크 스무딩 등.
5. **속도 목표(≤2× 실시간, L4) 달성 방법:** 모델 경량화, 증류(distillation), 양자화, 얼굴영역 저해상 처리, 배치 등.
6. **학습 방법론:** teacher-student 증류(고품질 diffusion 선생님 → 빠른 GAN 학생), few-shot 스타일 학습, from-scratch 페어 학습(pix2pix류) 중 이 목적에 최적은? 각 방식의 라이선스 안전성.
7. **오픈 라이선스 학습 데이터셋:** 얼굴 이미지(상업 사용 가능), 스타일 참조 이미지 확보 방안. (FFHQ 등 주요 얼굴 데이터셋의 라이선스도 확인)
8. **YOLOX 검출 결과 연동:** 검출된 bbox/랜드마크로 얼굴 정렬→스타일화→역합성할 때의 best practice, 경계 블렌딩(seamless) 기법.

## 5. 이미 조사된 내용 (재조사 말고 검증·보강만)

**오픈 라이선스 후보 (확인됨):**
- animegan2-pytorch — **MIT**, 실시간급 경량 GAN, 2.5D 반실사 룩. (현재 1순위 후보)
- DCT-Net — **Apache 2.0**, 초상 스타일화 few-shot, 실시간급.
- FLUX.1-schnell — **Apache 2.0**, 고품질 diffusion(느림, 오프라인 선생님용).
- Chroma1-HD — **Apache 2.0**, Flux-schnell 기반 아트/애니 특화(오프라인).
- Qwen-Image-Edit — **Apache 2.0**, 지시 기반 편집(오프라인).

**라이선스로 제외됨:**
- VToonify, FRESCO, Rerender-A-Video (S-Lab 비상업), AnimeGANv3(커스텀 비상업), SDXL/Illustrious/Pony "Anima" 체크포인트(OpenRAIL/Fair AI), DomoAI(상용 클로즈드).

**측정된 사실:**
- L4에서 Flux(4bit) img2img ≈ **10초/프레임** → 실시간 불가, 오프라인 데이터 생성용.
- animegan2 경량 GAN은 384~512 해상도에서 **1:2 통과**(1분 영상 43~87초).
- 잠정 아키텍처: **Flux/Chroma = 오프라인 선생님(스타일 데이터 생성)**, **경량 GAN = 실시간 학생(증류)**.
