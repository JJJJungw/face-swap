# 딥리서치 결과 — 오픈 라이선스 얼굴 카툰화 비식별화 파이프라인

> 104개 에이전트 다중 소스 조사 + 3표 적대적 검증. 2026-07-21.

---

## 🔑 가장 중요한 발견 2가지 (이게 방향을 바꿈)

**1. "카툰화 = 비식별화"가 아니다 — 스타일화만으론 비식별을 보장 못 함.**
- 스타일화하면 일반 얼굴인식(ArcFace) 매칭이 크게 떨어짐: TPR 0.765 → **0.372** (StyleID 벤치).
- **그러나** 스타일에 강건한 인식기(StyleID)는 완전히 처음 보는 스타일에도 **0.744 TPR로 재식별**함 → 신원이 스타일을 뚫고 살아남음.
- **결론: 카툰화에 더해 "명시적 신원 억제(identity-suppression) 손실/모듈"을 반드시 넣어야 비식별이 보장됨.** 이게 안 들어가면 "예쁘게 바뀐 같은 사람"이라 비식별 목적 실패.

**2. 코드+가중치+데이터 전부 Apache/MIT인 "완성형"은 존재하지 않음.**
- MIT/Apache로 확인되는 건 대부분 **"코드"에만** 해당. **가중치·학습데이터는 별도 라이선스**(FFHQ, StyleGAN2=NVIDIA 비상업, FDF=CC-NC, SD=OpenRAIL 등)로 막히는 경우가 많음.
- → **강점을 조합해 자체 학습**하는 게 불가피.

## ✅ 추천 아키텍처 (연구 결론)

> **(A) 오프라인 선생님:** Flux/Chroma(Apache)로 **신원을 치환(replace)하면서 2.5D 스타일을 입힌** (원본→비식별+카툰) 페어 데이터 대량 생성.
> **(B) 실시간 학생:** animegan2/DCT-Net 급 경량 GAN을 그 데이터로 증류 학습 → L4에서 1:2 통과.
> **(C) 비식별 보장:** 학생 학습 목표에 **신원 임베딩 억제 손실**(원본 얼굴 임베딩과 출력 임베딩의 코사인을 낮추도록)을 추가.

즉 이전 계획(선생님-학생 증류)은 맞았고, 여기에 **(C) 신원 억제 손실이 필수로 추가**되는 게 이번 리서치의 핵심 업데이트예요.

## 다음에 파야 할 오픈 질문 (이번 조사에서 미해결)

1. **상업 사용 가능 얼굴 데이터셋** 조합 (FFHQ는 Flickr CC/PD 혼합이라 개별확인 필요, FDF·CelebA-HQ 등).
2. **상업 가능한 얼굴인식 인코더** — 신원 억제 손실 구현에 쓸 FR 모델. ArcFace 가중치 라이선스 확인 필요, 대체 오픈 FR 있는가.
3. **오픈 라이선스 시간적 일관성 구현체** (optical flow smoothing / temporal loss) + L4 ≤2× 경량화·양자화 구체 구현.
4. **YOLOX bbox/랜드마크 → 정렬→스타일화→역합성** seamless 블렌딩(Poisson/feather) best practice, 프레임별 비식별 검증 임계값(cos<0.3 등).

## 정리 (한 문장)

**"오픈 라이선스로 실시간 2.5D 카툰 비식별화"의 완성품은 없다 → Flux/Chroma 선생님으로 (비식별+2.5D) 데이터를 만들어 경량 GAN 학생에 증류하되, 신원 억제 손실을 반드시 넣어 비식별을 보장한다.** 그리고 라이선스는 코드뿐 아니라 **가중치·데이터까지** 전부 확인해야 한다.
